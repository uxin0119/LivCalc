# SPOIL THE SERVER - Game Design Document

## 게임 콘셉트
로그/터미널 UI를 가진 로그라이크 + 덱빌딩 + 진행상황 축적형 해킹 게임

### 핵심 아이디어
- 플레이어는 해커가 되어 서버의 보안을 뚫고 기밀 정보를 탈취한다
- 로그 화면처럼 보이는 UI에서 명령어를 입력하여 익스플로잇 실행
- "Slay the Spire" + "Spoil the Server" - 서버를 망가뜨리며 올라간다
- 최소한의 플레이어 액션으로 게임이 자동 진행
- 스크립트를 작성하여 공격을 자동화

### 테마
- **플레이어**: 화이트햇/그레이햇 해커
- **적**: 서버 보안 시스템, 방화벽, IDS/IPS
- **프로세스(Process)**: 익스플로잇, 페이로드, 바이패스 기법
- **스레드(Threads)**: Thread Pool (스레드 풀)
- **무결성(Integrity)**: System Integrity (시스템 무결성)
- **방화벽(Firewall)**: Firewall (방화벽)

## 게임 메커니즘

### 1. 전투 시스템
- **Auto-battle with Commands**: 기본적으로 자동 전투, 핵심 순간에만 명령어 입력
- **명령어 기반**: attack, defend, heal, special, auto, skip
- **타이핑 효과**: 플레이어 입력이 실제 터미널처럼 보임

### 2. 프로세스 라이브러리
- **프로세스**: 각 프로세스는 실행 가능한 프로그램
  - `attack_heavy.exe` - 고처리량 공격
  - `attack_quick.sh` - 저사이클 공격
  - `firewall.bat` - 방화벽
  - `heal.py` - 회복
  - `scan.exe` - 정보 수집
  - `ddos.sh` - 멀티 어택

### 3. 스크립트 시스템
플레이어가 전투 로직을 미리 작성:
```
IF enemy.hp > 50 THEN defend
IF energy >= 3 THEN attack_heavy
ELSE attack_quick
IF self.hp < 30 THEN heal
```

### 4. 매크로 시스템
여러 명령을 하나로 묶어서 실행:
```
MACRO burst:
  1. attack_quick
  2. attack_quick
  3. attack_heavy
```

### 5. AI 자동화
- `auto` 명령으로 AI에게 전투 위임
- 플레이어는 로그만 관찰
- 언제든 수동 모드로 전환 가능

## 게임 루프

### 런 구조
1. **라이브러리 구성**: 프로세스 선택 및 라이브러리 구축
2. **노드 선택**: 맵에서 경로 선택
3. **전투**: 스크립트 실행 or 수동 명령
4. **보상**: 프로세스 획득 or 업그레이드
5. **반복**: 보스까지 진행

### 노드 타입
- **전투**: 일반 적 조우
- **엘리트**: 강한 적, 좋은 보상
- **이벤트**: 랜덤 이벤트
- **상점**: 카드 구매/업그레이드
- **휴식**: 체력 회복
- **보스**: 층 최종 보스

## 진행상황 축적 (메타 프로그레션)

### 레벨 시스템
- LV 1: 기본 명령어 (attack, defend)
- LV 3: 스크립트 에디터
- LV 5: 매크로 시스템
- LV 7: 고급 명령어
- LV 10: AI 자동 전투
- LV 15: 커스텀 명령어 생성

### 영구 업그레이드
- **RAM+**: 라이브러리 크기 증가
- **CPU_OC**: 스레드 재생 증가
- **SSD**: 프로세스 드로우 증가
- **Network**: 노드 선택지 증가
- **Backup**: 이전 런 프로세스 일부 복구

### 언락 시스템
- 새로운 프로세스 타입
- 새로운 적 타입
- 새로운 노드 타입
- 새로운 보스

## UI/UX 디자인

### 터미널 스타일
```
[00:23:15] SYSTEM BOOT... [OK]
[00:23:16] Loading deck... [████████████] 100%
[00:23:17] Entering node_1...
[00:23:18] ENEMY DETECTED: virus.exe [HP: 50]
> attack_
```

### 컬러 코딩
- **GREEN**: 성공, 회복, 시스템 메시지
- **RED**: 에러, 데미지, 적 행동
- **YELLOW**: 경고, 이벤트
- **CYAN**: 플레이어 입력, 정보
- **MAGENTA**: 특수 효과

### 애니메이션
- 타이핑 효과 (글자가 하나씩 나타남)
- 프로그레스 바 (`[████░░░░] 50%`)
- 커서 깜빡임
- 스크롤 효과

## 게임 밸런스

### 난이도 곡선
- 초반: 간단한 명령어로 승리 가능
- 중반: 스크립트 작성 권장
- 후반: 최적화된 덱과 스크립트 필요
- 엔드게임: AI 활용 or 고급 전략 필요

### 리워드 구조
- 전투: 골드 + 프로세스 선택
- 엘리트: 레어 프로세스 + 유물
- 보스: 강력한 유물 + 대량 골드
- 런 완료: 경험치 + 영구 업그레이드 포인트

## 확장 가능성

### Phase 1 (MVP)
- 기본 전투 시스템
- 5종 프로세스
- 3종 적
- 1개 보스
- 기본 메타 프로그레션

### Phase 2
- 스크립트 시스템
- 10종 프로세스
- 다양한 노드 타입
- 멀티플 보스

### Phase 3
- 매크로 시스템
- AI 자동화
- 고급 메타 프로그레션
- 엔드게임 콘텐츠

### Phase 4
- 커스텀 명령어 생성
- 멀티플레이어 리더보드
- 일일 챌린지
- 모드 시스템

## 기술 스택

### Frontend
- Next.js + React
- TypeScript
- Tailwind CSS (터미널 스타일링)

### 상태 관리
- React Context or Zustand
- 게임 상태: 현재 전투, 덱, HP, 에너지
- 메타 상태: 레벨, 언락, 영구 업그레이드

### 로컬 스토리지
- 진행상황 저장
- 메타 프로그레션 데이터
- 언락 정보

## 개발 원칙

### 모듈화 규칙
- **한 파일은 1000라인을 넘지 않는다**
- 초과 시 반드시 기능별로 분리
- 명확한 책임 분리

### 파일 구조
```
src/app/auto-log/
├── page.tsx              # 메인 게임 페이지
├── design.md             # 이 문서
├── components/
│   ├── Terminal.tsx      # 터미널 UI
│   ├── CommandInput.tsx  # 명령어 입력
│   ├── LogDisplay.tsx    # 로그 출력
│   ├── Combat.tsx        # 전투 화면
│   ├── DeckBuilder.tsx   # 덱 구성
│   └── NodeMap.tsx       # 노드 맵
├── game-logic/
│   ├── combat.ts         # 전투 로직
│   ├── cards.ts          # 프로세스 정의
│   ├── enemies.ts        # 적 정의
│   ├── script.ts         # 스크립트 시스템
│   └── progression.ts    # 메타 프로그레션
└── types/
    └── game.ts           # 타입 정의
```

## 개발 우선순위

1. **기본 터미널 UI** - 타이핑 효과, 로그 출력
2. **간단한 전투 시스템** - 턴제, 기본 명령어
3. **프로세스 시스템** - 5개 기본 프로세스
4. **적 AI** - 간단한 패턴
5. **라이브러리 빌딩** - 프로세스 선택/제거
6. **메타 프로그레션** - 레벨, 언락
7. **스크립트 시스템** - 자동화
8. **고급 기능** - 매크로, AI

## 참고 사항
- 텍스트 기반이므로 성능 부담 적음
- 애니메이션은 CSS로 최대한 처리
- 모바일 대응 고려 (세로 모드 터미널)
- 접근성: 키보드 단축키 지원
